====================
1. KIẾN TRÚC HIỆN TẠI
====================

Thiết bị:
- 3 ESP32 receiver: esp1, esp2, esp3.
- 3 ASUS/Nexmon receiver: asus1, asus2, asus3.
- 1 ASUS phát Wi-Fi.

Luồng dữ liệu:

ESP/ASUS
→ Collection
→ TCP JSON Lines
→ Windows Realtime HAR Server
→ xử lý buffer, chọn window, PCHIP, tạo tensor
→ HTTP POST sang WSL
→ Haar DWT + z-score + Mamba GPU
→ trả label/confidence về Windows
→ SQLite + WebSocket
→ Web Dashboard.

=====================
3. CẤU HÌNH BUFFER
=====================

Dự kiến/current config:

RAW_BUFFER_SIZE = 1200
WINDOW_SIZE = 1000
STEP_SIZE = 100

Có 6 deque riêng:

esp1, esp2, esp3
asus1, asus2, asus3

ESP và ASUS là hai group độc lập:

groups["esp"].pending
groups["asus"].pending

Không dùng chung pending.

Một group bắt đầu inference khi:

1. Cả 3 receiver cùng loại đủ ít nhất 1200 raw packet.
2. Group không đang inference.
3. Sau prediction trước, cả 3 receiver đều có thêm >= 100 packet.

=======================
4. CÁCH CHỌN WINDOW SEQ
=======================

Seq quay vòng 12-bit:

0 → 1 → ... → 4095 → 0

SEQ_MODULO = 4096
SEQ_HALF_RANGE = 2048

Logic mới mong muốn/current:

- Lấy seq cuối của 3 buffer.
- Chọn seq cuối “cũ nhất” theo modulo 4096,
  tức receiver chậm nhất.
- Gọi đó là end_seq chung.
- start_seq = end_seq - 999 theo modulo.
- Window vẫn theo chiều thời gian tăng dần:
  start_seq → ... → end_seq.

Ví dụ:

esp1 cuối = 2199
esp2 cuối = 2229
esp3 cuối = 2219

→ end_seq = 2199
→ start_seq = 1200
→ model dùng seq 1200 đến 2199.

=================
5. NỘI SUY PCHIP
=================

PCHIP chạy ở Windows preprocess.py.

- Nội suy amplitude, không nội suy I/Q raw.
- Packet thiếu/lỗi CSI sẽ thành missing.
- Dự kiến giới hạn:
  MAX_MISSING_RATIO = 0.02
  MAX_CONSECUTIVE_MISSING = 5

Với window 1000:

- tối đa 20 packet mất tổng cộng;
- tối đa 5 packet mất liên tiếp;
- mất đầu/cuối window thì bỏ window;
- mất quá giới hạn thì không inference.

=========================
9. NHÃN VÀ CONFIDENCE WEB
=========================

Model WSL trả sẵn JSON kiểu:

{
  "label_id": 7,
  "label": "Run",
  "probability": 0.9924
}

WSL map label_id sang tên nhãn bằng class_names bên trong esp.pt/asus.pt.

Windows/Web không cần map lại 7 → Run.

Web chỉ hiển thị trực tiếp:

Action:
- label, ví dụ Run.

Confidence:
- probability * 100, ví dụ 99.24%.

Lưu ý:
Confidence là độ tự tin của model cho window hiện tại,
không phải accuracy tổng thể của model.

HAR_LABELS bên Windows:

HAR_LABELS = (
    "Sit Down",
    "Stand Up",
    "Pick Up Object",
    "Fall",
    "Lie Still",
    "Walk",
    "Run",
    "Empty Room",
)

chỉ dùng khi mock/random fallback.
Khi model WSL chạy thật bình thường thì không dùng để map label.

=====================
10. LATENCY VÀ SQLITE
=====================

latency_ms hiện là tổng:

Windows preprocess
+ PCHIP
+ HTTP Windows → WSL
+ Haar/z-score
+ GPU Mamba
+ nhận response về Windows.

Không tính thời gian chờ thu đủ 1000 packet.

Khi inference thành công:

action = tên hành động
confidence = số 0..1
latency_ms = số ms

Khi model/WSL lỗi:

action = "predict_error"
confidence = NULL
latency_ms = NULL

SQLite table:

prediction_logs(
  id,
  web_updated_at,
  source,
  input_timestamp,
  latency_ms,
  action,
  confidence
)

12. MỤC TIÊU CHAT HIỆN TẠI
========================
